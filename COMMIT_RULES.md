# コミットルール

## フォーマット

```text
[Prefix] コンテストID-問題: 説明
```

## Prefix

| Prefix       | 用途                      | 例                                        |
| ------------ | ------------------------- | ----------------------------------------- |
| `[AC]`       | コンテスト中/終了時の正解 | `[AC] abc430-a: 実装完了`                 |
| `[Upsolve]`  | コンテスト後の解き直し    | `[Upsolve] abc430-e: editorial参考に実装` |
| `[WIP]`      | 未完成・思考記録          | `[WIP] abc430-c: サンプル2が通らない`     |
| `[CONTEST]`  | コンテスト参加記録        | `[CONTEST] abc430: 参加 (0完)`            |
| `[REFACTOR]` | リファクタリング・別解    | `[REFACTOR] abc430-a: より簡潔に書き直し` |
| `[FIX]`      | バグ修正                  | `[FIX] abc430-c: オーバーフロー対策`      |
| `[CHORE]`    | 環境設定・雑務            | `[CHORE] toolchain更新`                   |

## Tips

**複数問題**: `[AC] abc430-a,b,c: A~C完了`  
**コンテスト全体（AC有）**: `[AC] abc430: 終了 (4完)`  
**コンテスト全体（AC無）**: `[CONTEST] abc430: 参加 (0完)` または `[WIP] abc430: 0完`  
**WIPには本文に考察を残す**:

```text
[WIP] abc430-e: DP遷移が分からない

dp[i][j]で定義したが復元方法が不明
次: editorialを読む
```
